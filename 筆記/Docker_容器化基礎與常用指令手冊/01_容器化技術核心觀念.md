## 第一章：容器化技術核心觀念

### 1.1 什麼是容器化 (Containerization)？

容器化是一種輕量級的虛擬化技術。它允許開發者將應用程式及其所有依賴項目（如函式庫、設定檔、環境變數）打包在一個隔離的單位中，稱為**容器**。

### 1.2 虛擬機 (VM) vs. 容器 (Container)

在資工領域中，理解這兩者的差異是核心功課。

| 特性 | 虛擬機 (VM) | 容器 (Container) |
| --- | --- | --- |
| **隔離層級** | 硬體級別虛擬化 | 作業系統級別虛擬化 |
| **作業系統** | 包含完整的 Guest OS | **共享** 宿主機 (Host) 的 Kernel |
| **啟動速度** | 分鐘級 (需引導 OS) | **秒級** (僅啟動進程) |
| **資源消耗** | 佔用大量內存與硬碟 | 極輕量，僅需必要檔案 |
| **效能** | 有虛擬化損耗 | 幾乎接近原生效能 |

### 1.3 為什麼我們需要 Docker？

對於像 `dog-project` 這樣的前後端專案，Docker 解決了以下痛點：

* **環境一致性 (Environment Consistency)**：解決「在我的電腦可以跑，但部署到伺服器就壞掉」的問題。無論在 Windows、Mac 還是 Linux，執行環境完全相同。
* **隔離性 (Isolation)**：前端 Node.js 與後端 Python 環境互不干擾。
* **快速擴展 (Scalability)**：當流量增加時，可以快速複製出多個相同的容器來分擔負載。
* **微服務架構的基礎**：讓前後端能獨立打包，卻能透過 Docker Network 輕鬆溝通。
